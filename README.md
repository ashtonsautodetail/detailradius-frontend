# DetailRadius — Frontend

The complete DetailRadius website: a single-page app (`index.html`, all inline JS/CSS) plus static pages, deployed to Netlify Drop at https://serene-cupcake-78a254.netlify.app

## Deploying

Netlify Drop replaces the WHOLE site each deploy, so always deploy the full file set:

1. Zip everything except the `make_*.py` generators and this README:
   `zip -j detailradius-site.zip index.html robots.txt sitemap.xml manifest.webmanifest sw.js icon-*.png apple-touch-icon.png mobile-detailing-*.html privacy.html terms.html fleet.html for-detailers.html llms.txt`
2. Netlify → serene-cupcake-78a254 → Deploys → drag the zip (or "browse files to upload").
3. **Bump `VERSION` in `sw.js`** (dr-v4 → dr-v5 …) whenever `index.html` changes, so installed PWA clients drop their old cache.

## What's what

- `index.html` — the entire app: customer search/booking, detailer dashboard (incl. Pro tier), owner tab (referral ledger, fleet inquiries), auth, PWA install banner. Talks to Supabase (`asfqmjccfnjfrspvhnpt`) and the Netlify-functions backend (`detailradius-backend` repo).
- `mobile-detailing-*-il.html` — 10 city SEO pages with live detailer listings (generated by `make_city_pages.py`; edit the template there, rerun, re-zip). **Note:** the generator rebuilds `sitemap.xml` with only home+city URLs — re-add privacy/terms/fleet/for-detailers entries after running it.
- `fleet.html` — B2B/fleet inquiry page (writes to `fleet_inquiries` table).
- `for-detailers.html` — detailer recruitment landing.
- `sw.js` + `manifest.webmanifest` + icons — the PWA layer. The service worker is network-first and never caches Supabase/Stripe/backend traffic.
- `llms.txt` — AI-assistant discovery file.
- `make_legal_pages.py`, `make_icons.py` — generators for legal pages and icon PNGs.
